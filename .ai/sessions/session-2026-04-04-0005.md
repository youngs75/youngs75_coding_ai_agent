#### 11-1. 훅 시스템 (`core/hooks.py`, `core/builtin_hooks.py`)
- **HookManager**: 이벤트 기반 훅 매니저 (동기/비동기 핸들러 모두 지원)
- **7개 이벤트**: PRE/POST_TOOL_CALL, PRE/POST_NODE, ON_ERROR, PRE/POST_LLM_CALL
- **HookContext**: 핸들러 체이닝 — pre 훅에서 tool_args 수정 시 실제 실행에 반영
- **우선순위 기반 실행**, 에러 격리 (개별 핸들러 실패 시에도 계속), 취소 지원 (cancel=True)
- **내장 훅 3종**: logging_hook, timing_hook, audit_hook
- **YAML/JSON 설정 로드**: `.agent/hooks.yaml`에서 훅 자동 등록
- **ParallelToolExecutor 연동**: `_execute_single_tool()`에 PRE/POST 훅 호출 추가
- **BaseGraphAgent 연동**: `_wrap_node()` 메서드로 PRE_NODE/POST_NODE/ON_ERROR 자동 발행
- 테스트: **53개** (12개 테스트 클래스)

#### 11-2. Coordinator Mode (`agents/orchestrator/coordinator.py`, `task_graph.py`)
- **CoordinatorMode**: 복합 작업 → 서브태스크 분해 → 병렬 워커 실행 → 결과 통합
- **TaskGraph**: DAG 기반 서브태스크 의존성 관리 (Kahn's 알고리즘 순환 감지)
- **웨이브 실행**: 위상 정렬 기반 실행 웨이브 — 웨이브 내 병렬, 웨이브 간 순차
- **순환 복원력**: 순환 감지 시 의존성 해제 후 단일 병렬 웨이브로 실행
- **부분 실패 허용**: 워커 실패/타임아웃 시에도 결과 통합 (LLM이 실패 사실 인지)
- **효율성 메트릭**: `parallel_efficiency = 실제시간 / 개별시간합`
- **Orchestrator 그래프 확장**: classify에서 `__coordinator__` 라우팅 → coordinate 노드
- 스키마: SubTask, WorkerResult, CoordinatorResult TypedDict 추가
- 테스트: **41개** (11개 테스트 클래스)
