## 이번 세션 작업 내역

### 1. 3종 Coding Agent 비교 분석 (Claude Code, Codex, DeepAgents)
- TeamCreate로 3개 에이전트를 병렬 스폰하여 소스코드 분석
- 핵심 발견: Claude Code/Codex는 **LLM 1개 + 도구 + 단순 루프**, 우리는 12+ 노드/4+ LLM 호출
- 3종 모두 `__pycache__` 자동 정리 없음, 진동 감지 없음, 에러 분류 없음
- 우리 Harness가 에러 분류/수정 가이드 면에서 가장 발전했지만, 오히려 SLM을 혼란시킴

### 2. Langfuse 트레이싱으로 근본 원인 분석
- **DashScope 400 에러**: `Can only get item pairs from a mapping` — AIMessage tool_calls 직렬화 문제
- **테스트 파일 미생성**: SLM이 write_file 대신 read_file만 반복 → StallDetector FORCE_EXIT
- **planned_files 오판**: 디스크에 있는 파일을 written_files에 없다고 "누락" 판정 → SLM 혼란
- **에러 분류 "Unknown"**: regex 패턴에 "파일 누락"이 매칭 안 됨
- **VERIFY rubber stamp**: 약한 모델이 빈 결과도 "passed=True"
- **iteration 이중 증가**: VERIFY + RUN_TESTS 각각 +1

### 3. 핵심 아키텍처 결정
- **원칙**: Harness = 기계적 도구 제공자, LLM = 판단자
- SLM/vLLM 환경에서 Harness가 판단(에러 분류, 코드 검증)을 대신하면 안 됨
- 에러 원문만 전달하고 LLM이 직접 판단하게 해야 함

### 4. 파이프라인 단순화 구현 (10노드→4노드)
- **agent.py**: NODE_NAMES + init_nodes + init_edges 재작성
- **agent.py**: `_generate_code()` 신규 — 기존 GENERATE_FINAL + 정적 검증 + 마크다운 폴백 + 반복 감지 통합
- **agent.py**: `_inject_error()` 신규 — 에러 원문만 전달, 분류/힌트/가이던스 전부 제거
- **agent.py**: `_retrieve_memory()` — PARSE의 안전장치 초기화 흡수
- **agent.py**: `_run_tests()` — planned_files를 디스크 존재 확인으로 버그 수정
- **agent.py**: `_sanitize_ai_tool_calls()` — DashScope/vLLM tool_calls 직렬화 호환
- **agent.py**: `_clean_python_cache()` — 테스트 전 __pycache__ 자동 정리
- **schemas.py**: file_manifest 레거시 주석
- **test_coding_assistant.py**: 노드명 + 반환값 테스트 업데이트

### 5. 이전 세션(#0006)의 개선사항도 포함
