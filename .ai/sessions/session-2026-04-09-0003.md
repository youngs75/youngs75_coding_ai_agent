## 중요 참고사항

### 미들웨어 체인 아키텍처
- 기존 AgentMiddleware 양파 패턴을 유지하여 호환성 보장
- ResilienceMiddleware는 체인 가장 바깥쪽(첫 번째)에 배치
- 순서: Resilience → Window → Summarization → Memory
- dead code였던 FailureMatrix, RetryWithBackoff, ModelFallbackChain, AbortController 모두 live 상태

### SubAgent 메모리 접근
- worker.py에서 workspace `.ai/memory/` 경로로 MemoryStore 자동 초기화
- 모든 에이전트 타입(coding_assistant, deep_research, simple_react)에서 메모리 접근 가능

### Planner 연동
- _detect_complexity()로 복잡도 판단 → complex 요청만 PlannerAgent 호출
- 단순 요청은 기존 경로(delegate 직행) 유지

### SLM 활용 지점
- Orchestrator classify: FAST (parsing)
- CodingAssistant parse_request: FAST (parsing)
- CodingAssistant execute_code: FAST (tool_planning)
- MemoryMiddleware 자동 축적: FAST (판단용)
