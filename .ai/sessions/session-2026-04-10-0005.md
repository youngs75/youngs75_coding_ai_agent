### 2. Groundhog Day 루프 근본 원인 분석

#### 원인 1: write_file content가 대화 히스토리에 잔류
- LLM이 write_file(content="전체 파일 내용...") 호출 → AIMessage에 파일 전체 내용이 남음
- 파일 4-5개 생성 시 tool_call args만 30-40K 토큰 → 100K 초과
- MessageWindowMiddleware(100K) 발동 → 최근 8개 메시지만 남기고 전부 삭제
- LLM이 이전 작업을 잊어버리고 처음부터 재시작

#### 원인 2: 시스템 프롬프트의 체크리스트가 업데이트 안 됨
- planned_files 체크리스트가 항상 `- [ ]` 미체크 상태 (루프 밖에서 1회만 빌드)
- written_files로 `[x]` 업데이트하는 로직 없음

#### 원인 3: 미들웨어 적용 방식 차이 (우리 vs 3종 에이전트)
- 3종 에이전트: Summarization은 context window 85% 도달 시에만 → 평소 전체 히스토리 보존
- 우리: MessageWindow(100K)가 매 iteration마다 실행 → 도구 결과가 잘려나감
