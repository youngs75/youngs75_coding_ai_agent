### 3. Human-in-the-loop (HITL) 계획 승인
Planner Agent가 구현 계획을 수립하면 사용자 승인을 받은 후 실행하는 패턴 구현.

#### 구현 방식
- `astream_events`는 `GraphInterrupt` exception을 전파하지 않음 (검증 완료)
- **`aget_state()` 기반 interrupt 감지** 방식 채택:
  1. 이벤트 스트림 종료 후 `agent.graph.aget_state(config)` 호출
  2. `state.next`가 비어있지 않고 `task.interrupts`가 있으면 interrupt 감지
  3. 계획을 Panel로 표시 → `ask_plan_approval()` → 승인 시 `Command(resume=True)` 재개

#### 변경 파일
- `agents/orchestrator/agent.py`: `plan` 노드에 `interrupt(plan_text)` 추가, classify 프롬프트 개선
- `agents/orchestrator/config.py`: coding_assistant description 강화 (풀스택 명시)
- `cli/renderer.py`: `show_plan()`, `ask_plan_approval()` 메서드 추가
- `cli/app.py`: while 루프 + `aget_state()` 기반 HITL 처리, `Command` import
