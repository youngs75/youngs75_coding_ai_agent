### Phase 3: Structured Output + MCP 검증 도구 도입

DeepAgents(`response_format` + `ToolStrategy(schema=Model)`)와 Codex(`output_schema` + `strict: true`)를 참고하여 4가지 구조적 제어 도입.

#### 1. Pydantic 스키마 모델 (schemas.py)
- `ParseResultModel`: task_type, language, framework, description, target_files, requirements
- `VerifyResultModel`: passed, issues, suggestions
- `FileManifest` + `FunctionContract` + `SharedObject`: 파일 간 인터페이스 선언

#### 2. _parse_request Structured Output
- `with_structured_output(ParseResultModel)` → 스키마 강제
- 미지원 모델 시 기존 `json.loads` 폴백

#### 3. _verify_result Structured Output
- `with_structured_output(VerifyResultModel)` → JSON 파싱 실패 시 `passed=True` 우회 제거
- 미지원 모델 시 기존 방식 폴백

#### 4. _plan_file_manifest 노드 (신규)
- 코드 생성 전 FileManifest를 Pydantic으로 선언
- FAST 모델로 파일 간 함수 계약/공유 객체를 사전 선언
- `_generate_final`에서 manifest를 시스템 프롬프트에 주입
- 그래프 흐름: `EXECUTE → PLAN_MANIFEST → GENERATE_FINAL → VERIFY`

#### 5. MCP validate_consistency 도구 (server.py)
- AST 기반 정적 분석: 함수 시그니처 불일치, import 미존재, SyntaxError 감지
- `_generate_final`의 도구 바인딩에 자동 포함
- LLM이 코드 작성 후 자발적 호출 가능
