## 다음 단계 (필수 — 아키텍처 재설계)

### 1. `_run_tests` 노드 재설계 (최우선)
**현재 문제**: Harness가 venv 생성 + pip install + pytest 실행을 LLM 모르게 수행
→ LLM은 프로젝트 구조를 알지만 Harness는 모름 → 경로/패턴 불일치 시 해결 불가

**3종 에이전트 패턴**: LLM이 shell 도구로 직접 `pip install`, `pytest` 실행
→ LLM이 프로젝트 구조를 이해하므로 경로 문제를 스스로 해결

**개선 방향**:
- `_run_tests` 내부의 자동 venv/설치/실행 → LLM에게 shell/execute 도구를 제공하여 직접 실행하도록 전환
- Harness는 "sandbox 제공자" 역할만 (Docker 격리, 타임아웃 제한)
- 에러 분석도 regex 기반 → LLM 직접 판단으로 전환
- `_inject_error`의 regex 패턴 매칭 → 에러 원문을 그대로 LLM에 전달 (이미 부분적으로 적용됨)
